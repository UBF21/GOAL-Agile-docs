# Level 3 — GOAL Program Lead (GPL)

## Overview

The GOAL Program Lead is the highest practitioner-level certification in the GOAL system. It validates competency in operating the GOAL framework at the **program level** — specifically, coordinating multiple teams operating GOAL simultaneously, managing cross-team dependencies, and leading program-level events and governance structures.

The GPL corresponds to the Program Flow Master and Program Strategist roles defined in GOAL's multi-team scaling model. It is designed for senior practitioners who are responsible not for a single team's flow, but for the health of the entire delivery program across multiple teams.

---

## Target Audience

The GPL certification is appropriate for:

- Program Flow Masters managing 2–8 GOAL teams
- Program Strategists responsible for cross-team goal alignment and program-level value delivery
- Senior Agile coaches or transformation leads operating at the program level
- Engineering directors, delivery managers, or VP-level practitioners in GOAL-scaled environments
- Consultants specializing in large-scale agile transformation using GOAL

---

## Prerequisites

To be eligible to sit the GPL examination, candidates must meet ALL of the following:

**Prior certification:** Active GFMC certification (not expired at time of GPL application)

**Multi-team experience:** A minimum of twelve (12) months of experience working across multiple teams in a coordination role. This experience must involve actual cross-team dependency management and program-level planning — not simply managing a single large team.

**Program cycles:** A minimum of four (4) complete program cycles coordinated at the program level. A "program cycle" is defined as one complete cycle where the candidate was responsible for facilitating the Program Sync, maintaining the Dependency Register, and co-facilitating the Program Retrospective for a group of at least two GOAL teams.

**Program Portfolio:** Candidates must submit and have accepted a Program Portfolio before sitting the examination.

## Knowledge Domains

| Domain | Description | Exam Weight |
|--------|-------------|:-----------:|
| Domain 1: Program GOAL Architecture | Scaling tiers, program-level artifacts, multi-team structures | 15% |
| Domain 2: Program-Level Roles | Program Flow Master and Program Strategist authority, boundaries, and execution | 15% |
| Domain 3: Dependency Management | Identification, tracking, escalation, and structural resolution of cross-team dependencies | 25% |
| Domain 4: Program-Level Metrics | Cross-Team Block Rate, Dependency Rate, Program Goal Completion Rate, interpretation | 20% |
| Domain 5: Multi-Team Coordination Events | Program Sync, Cross-Team Dependency Review, Program Retrospective — facilitation and execution | 15% |
| Domain 6: Program Anti-Patterns and Scale Limits | Recognizing scaling anti-patterns, knowing when GOAL's scaling model reaches its limits | 10% |

### Domain Highlights

**Domain 3 — Dependency Management (25% — highest weighted)**

The core skill that differentiates a GPL practitioner:
- Identifying dependencies before they become blockers using the Dependency Register
- Applying status transitions (Identified → Active → At Risk → Blocked → Resolved) correctly
- Executing the dependency escalation protocol including when to involve the Program Strategist
- Calculating and interpreting the Dependency Rate metric
- Identifying when dependency patterns indicate an architectural coupling problem
- Making contingency decisions when a dependency blocks a primary goal

**Domain 4 — Program-Level Metrics (20%)**

- Cross-Team Block Rate: formula, target, and interpretation
- Average Dependency Resolution Time: target and what exceeding it indicates
- Dependency Rate: formula, target, and architectural implications
- Program Goal Completion Rate: formula and what drives it
- Program Cycle Accuracy Index: same formula as team-level, applied to program-level goals
- Reading program metric dashboards and identifying improvement opportunities

## Program Portfolio (Practical Requirement)

The Program Portfolio is a structured evidence submission required before sitting the examination.

**Section A — Program Cycle Records (required)**
Summaries of a minimum of 3 of the 4 required program cycles, each including:
- Teams involved (count and approximate size)
- Program Goals for the cycle
- Dependency Register summary (count of dependencies, resolved, blocked)
- Cross-Team Block Rate and average resolution time
- One significant cross-team coordination challenge and how it was resolved
- One program-level improvement from the Program Retrospective and its outcome

Each summary: 400–600 words.

**Section B — Dependency Escalation Evidence (required)**
Documentation of at least 2 real situations in which a cross-team dependency escalated to At Risk or Blocked, including:
- The dependency description
- The escalation path followed
- The resolution outcome
- What structural change (if any) was made to prevent recurrence

Each account: 400–600 words.

**Section C — Program Metrics Analysis (required)**
A written analysis of a real program's cross-team metrics over at minimum 3 consecutive program cycles:
- Metrics for each cycle (Cross-Team Block Rate, Dependency Rate, Program Goal Completion)
- Pattern identification and interpretation
- One specific improvement from program-level data and whether it was effective

Length: 800–1,200 words.

**Section D — Program Retrospective Evidence (required)**
Summary of a Program Retrospective facilitated by the candidate:
- Metrics reviewed
- Improvements identified
- How the improvements were tracked

Length: 400–500 words.

## Assessment Format

**Written Examination:**

| Component | Detail |
|-----------|--------|
| Exam type | Computer-based, proctored, closed-book |
| Number of questions | 130 |
| Duration | 180 minutes |
| Multiple choice (1 correct) | 80 questions |
| Multiple select (2+ correct) | 35 questions |
| Scenario-based with analysis | 15 questions |

**Panel Review (after passing written exam):**

The GPL Panel Review is a 60-minute live assessment conducted by a panel of two certified GPLs. The candidate presents their most significant program-level case (from the Program Portfolio) and defends their decisions and approaches under questioning.

The Panel Review evaluates:
1. Depth of understanding of program-level GOAL mechanics
2. Quality of judgment in ambiguous dependency and escalation scenarios
3. Ability to distinguish between GOAL's scaling capacity and its limits
4. Communication and facilitation competency at the executive level

The Panel Review is pass/fail, evaluated against a published rubric.

## Passing Criteria

| Component | Minimum Required |
|-----------|:----------------:|
| Overall examination score | 80% (104 of 130 questions) |
| All domains | 70% minimum per domain |
| Panel Review | Pass (rubric evaluation) |
| Program Portfolio | Accepted (prior to exam) |

## Certification Validity and Renewal

**Validity:** 2 years

**Renewal:** 40 CEUs within 2-year period, with at least 20 CEUs from GPL-designated content (multi-team coordination, program metrics, scaling architecture). Must submit an Activity Summary confirming continued active program-level practice.

**Expired:** Must re-sit full GPL examination and resubmit an updated Program Portfolio. GFMC credential is not affected by GPL expiration.

---

## What GPL Holders Are Authorized to Do

In addition to all GP and GFMC authorizations:

- Serve as panelists in GPL Panel Reviews
- Evaluate GFMC case studies as a reviewer
- Formally attest to experience prerequisites for GFMC and GPL candidates
- Apply for participation in the Institute's GPL Advisory Network
- Be listed as a recommended GOAL program-level consultant on the Institute's practitioner directory
- Contribute to GOAL methodology evolution through the Institute's practitioner feedback program
