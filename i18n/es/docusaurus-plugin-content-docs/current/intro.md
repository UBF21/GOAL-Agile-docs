# ¿Qué es GOAL?

**GOAL** (Goal-Oriented Adaptive Lifecycle — Ciclo de Vida Adaptativo Orientado a Objetivos) es un framework ágil diseñado para equipos modernos de desarrollo de software. Fue creado para resolver los problemas operativos concretos que Scrum y Kanban dejan sin resolver en entornos de TI del mundo real.

GOAL se estructura alrededor de una idea fundacional única:

> **Los objetivos son el contrato. Las tareas son el camino.**

Esto significa que los equipos se comprometen con resultados, no con listas de tareas específicas. El camino para alcanzar un objetivo puede cambiar. El objetivo en sí permanece estable dentro de un ciclo.

---

## Lo que GOAL introduce

| Concepto | Descripción |
|---------|-------------|
| Ciclos de Objetivos Adaptativos | Ciclos con límite de tiempo que pueden cerrarse anticipadamente cuando se cumplen los objetivos |
| Framework de Valor de 3 Capas | Definición concreta de qué significa "valor entregado" |
| Seguimiento de Eficiencia de Flujo | Mide el tiempo productivo frente al tiempo de espera en el proceso |
| Protocolo de Gestión de Interrupciones | Sistema formal para manejar trabajo urgente sin destruir el enfoque |
| Protocolo de Tareas Bloqueadas | Modelo de escalación con responsabilidad definida y plazos |
| Límites de WIP Dinámicos | Control del trabajo en progreso basado en fórmula, vinculado al tamaño del equipo |
| Definition of Done (DoD) | Criterio de calidad de dos niveles para cada tarea |
| Backlog Grooming Session | Evento dedicado al mantenimiento de la salud del backlog |
| Retrospectivas Basadas en Datos | Retrospectivas fundamentadas en métricas de flujo, no en opiniones |

---

## Objetivos principales

- Predictibilidad sin rigidez
- Entrega continua con enfoque estratégico
- Medición real del rendimiento que reemplaza las estimaciones en story points
- Reducción del agotamiento de los desarrolladores mediante el control de la carga cognitiva
- Visibilidad transparente del proyecto para los stakeholders

---

## Estructura del framework

GOAL consta de seis componentes estructurales: roles, artefactos, eventos, métricas, protocolos y reglas.

### Roles

- **Flow Master** — Responsable de la salud del proceso de entrega
- **Product Strategist** — Responsable de la dirección de valor del equipo
- **Delivery Team** — Grupo autoorganizado que ejecuta el trabajo
- **Backlog Curator** — Función del Product Strategist para mantener el backlog

### Artefactos

- Goal Cycle Plan
- Flow Board
- Flow Metrics Dashboard
- Block Register
- Definition of Done
- Cycle Accuracy Index

### Eventos

- Smart Planning Session
- Daily Flow Sync
- Backlog Grooming Session
- Goal Review
- Retrospectiva Basada en Datos

## Comparación rápida

| Dimensión | Scrum | Kanban | GOAL |
|-----------|-------|--------|------|
| Estructura de ciclo | Sprint fijo (1–4 semanas) | Sin ciclos | Ciclo de Objetivos Adaptativo (1–3 semanas) |
| Terminación del ciclo | Calendario | N/A | Objetivos completados |
| Flexibilidad de alcance | Fijo durante el sprint | Totalmente flexible | Tareas flexibles, objetivos fijos |
| Métrica principal | Velocidad | Throughput | Eficiencia de Flujo + Índice de Precisión de Ciclos |
| Manejo de interrupciones | Ruptura de sprint | En cualquier momento | Protocolo clasificado P1/P2/P3 |
| Deuda técnica | Generalmente ignorada | Visible si se rastrea | Trabajo de primera clase (10–20% asignado) |
| Medición de valor | No definida | No definida | Framework de Valor de 3 Capas |
